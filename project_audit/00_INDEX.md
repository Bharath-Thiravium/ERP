## What this project is (one line)

A **multi-tenant SaaS ERP / "modern SAP"** for Indian businesses (GST-aware) covering
Finance, HR, Inventory, CRM, Reports and Analytics — Django REST backend, React 19 SPA
frontend, a React Native employee-attendance mobile app, and a separate "master admin"
export bundle.

## Quick facts

- **Backend:** Django 5.2.9 + Django REST Framework, 13 local apps, ~98k LOC Python.
- **Frontend:** React 19 + Vite 7 + TypeScript + Zustand + TanStack Query + Tailwind + Ant Design, ~127k LOC.
- **Mobile:** React Native 0.81 (`EmployeeAttendanceApp`) with face/vision-camera attendance.
- **Database:** PostgreSQL (`modernsap`). ~100+ models across apps.
- **Async:** Celery + Celery Beat + Redis (broker, cache, channels layer).
- **Realtime:** Django Channels (WebSockets) over Redis.
- **Auth:** Dual system — SimpleJWT (master admin & company users) **and** a custom
  session-key auth for service users.
- **Deploy:** GitHub Actions → SSH to a single VPS, gunicorn + nginx + systemd. **No Docker.**

## Top findings (see report 11 for full risk assessment)

1. 🔴 **Committed secrets / data** — a 4.1 MB `modernsap_backup.sql` production dump, a
   hard-coded `EMAIL_ENCRYPTION_KEY` Fernet key default in `settings.py`, default DB
   password `mango`, and `backend/PROJECT_CREDENTIALS_SUMMARY.md` are all in the repo.
2. 🟠 **Repository hygiene** — 150+ status/fix `.md` files and dozens of one-off
   `fix_*.py` / `*.sh` scripts at the root make the real codebase hard to navigate.
3. 🟠 **Very large modules** — `finance/views.py` (5,103 lines), `authentication/views.py`
   (4,633), `finance/serializers.py` (4,278); frontend single files >1,900 lines.
4. 🟡 **Heavy ML dependencies** — `torch`, `transformers`, `dlib`, `face-recognition`,
   `sentence-transformers` make a clean install slow/heavy; tiered requirements files exist.
5. 🟡 **No containerization** — local & prod setup is script-driven and host-specific.
